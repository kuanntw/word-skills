# QA 檢查清單

## 產出前檢查

在開始產出文件之前，確認以下資訊是否齊全：

- [ ] 是否知道產品名稱？
- [ ] 是否知道文件名稱？
- [ ] 是否知道版本（如 V1.0.0）？
- [ ] 是否知道日期（如 2024/01/01）？
- [ ] 是否知道作者？
- [ ] 是否知道文件類型（規格書 / API 文件 / 操作手冊 / 驗收文件 / 教育訓練 / 專案文件）？
- [ ] 是否有章節內容？
- [ ] 是否有修訂紀錄資料？

## 產出中檢查

在產出文件過程中，確認以下事項：

### 標題
- [ ] 標題是否使用 Heading 1 到 Heading 9？
- [ ] 是否沒有使用 Normal 模擬標題？
- [ ] 是否沒有手動輸入標題編號後套 Normal？

### 內文
- [ ] 內文是否使用 _內文-縮排 或 _內文-無縮排？
- [ ] 內文字型是否為微軟正黑體？
- [ ] 內文字級是否為 12 pt？

### 清單
- [ ] 清單是否使用正確層級？
- [ ] 清單類型（字母/國字/符號/數字）是否符合用途？
- [ ] 清單下方延伸段落是否使用清單內文樣式？

### 程式碼
- [ ] 程式碼是否使用程式碼區塊（程式碼-有編號 或 程式碼-無編號）？
- [ ] 程式碼字型是否為 Courier New？
- [ ] 程式碼是否有灰底與框線？

### 表格
- [ ] 表格是否有表標號（表_標號）？
- [ ] 表標號是否置於表格上方並置中？
- [ ] 表頭是否有灰底？
- [ ] 表格是否有黑色細框線？

### 圖片
- [ ] 圖片是否有圖標號（圖_標號）？
- [ ] 圖標號是否置於圖片下方並置中？

### 附錄
- [ ] 附錄標題是否使用附錄格式（附錄A. 附錄標題）？

### 封面
- [ ] 封面是否包含產品名稱、文件名稱、版本、日期？
- [ ] 封面是否包含 Template V1.1.0？

## 產出後檢查

產出 .docx 後，執行以下檢查：

- [ ] 是否執行 `inspect_docx.py`？
- [ ] 是否沒有 `Error: Reference source not found`？
- [ ] 是否提醒使用者開啟 Word 後更新欄位（Ctrl+A, F9）？
- [ ] 是否確認目錄需要更新？
- [ ] 是否確認表目錄需要更新？
- [ ] 是否確認圖目錄需要更新？
- [ ] 是否確認表格沒有爆版（寬度超出頁面）？
- [ ] 是否確認頁首頁尾正常（黑線存在）？
- [ ] 是否確認標題字型為標楷體、內文字型為微軟正黑體？
- [ ] 是否確認標題與內文文字顏色為 #000000？
- [ ] 是否確認修訂紀錄表存在？

## 常見問題處理

| 問題 | 處理方式 |
|-----|---------|
| 目錄顯示「錯誤！找不到參考來源」 | 在 Word 中全選後按 F9 更新所有欄位 |
| 表格爆版 | 調整表格欄寬或使用自動調整 |
| 字型不正確 | 確認系統已安裝微軟正黑體，或文件使用內嵌字型 |
| 頁碼從封面開始 | 確認 section break 設定正確，正文獨立為一個 section |
