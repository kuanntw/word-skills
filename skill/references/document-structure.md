# 文件結構指南

以下為常見文件類型的建議骨架。產出文件時，請依據文件類型選擇對應章節結構。

## A. 系統規格書

1. 文件目的
2. 系統範圍
3. 名詞定義
4. 系統架構
5. 功能需求
6. 非功能需求
7. 權限設計
8. API 設計
9. 資料庫設計
10. 部署架構
11. 維運與監控
12. 附錄

### 章節說明

- **文件目的**：說明本文件目的、適用範圍、目標讀者。
- **系統範圍**：界定系統邊界，包含與不包含的功能。
- **名詞定義**：定義文件中使用的專有名詞與縮寫。
- **系統架構**：以圖表與文字描述系統整體架構、技術棧、模組關係。
- **功能需求**：以表格或條列方式列出各項功能需求，含優先級、輸入輸出、驗收標準。
- **非功能需求**：效能、安全性、可用性、擴充性等需求。
- **權限設計**：角色權限矩陣，以表格呈現。
- **API 設計**：各 API endpoint 詳細規格，使用表格呈現。
- **資料庫設計**：ERD、資料表結構、索引設計。
- **部署架構**：部署環境、網路拓樸、伺服器規格。
- **維運與監控**：日誌、監控、備份、災難復原。
- **附錄**：參考資料、術語對照表、補充說明。

## B. API 文件

1. 文件目的
2. 認證方式
3. 共通格式
4. Endpoint 一覽
5. API 詳細說明
6. 錯誤碼
7. 範例
8. 附錄

### 章節說明

- **文件目的**：API 文件目的、版本、基礎 URL。
- **認證方式**：Token、API Key、OAuth 等認證機制說明。
- **共通格式**：Request/Response 共通格式、Header、分頁格式。
- **Endpoint 一覽**：以表格列出所有 endpoint。
- **API 詳細說明**：每個 API 的 method、URL、參數、response、範例。
- **錯誤碼**：以表格列出所有錯誤碼與說明。
- **範例**：完整的使用情境範例。
- **附錄**：SDK 使用說明、變更紀錄。

## C. 操作手冊

1. 文件目的
2. 適用對象
3. 操作前準備
4. 操作流程
5. 常見問題
6. 異常排除
7. 附錄

### 章節說明

- **文件目的**：本手冊目的與適用範圍。
- **適用對象**：系統角色與對應權限。
- **操作前準備**：環境需求、帳號申請、軟體安裝。
- **操作流程**：以數字清單呈現逐步操作步驟，搭配螢幕截圖（使用圖片 placeholder）。
- **常見問題**：FAQ 格式，以表格呈現。
- **異常排除**：錯誤訊息與對應處理方式。
- **附錄**：快速參考卡、快捷鍵、聯絡資訊。

## D. 驗收文件

1. 驗收目的
2. 驗收範圍
3. 驗收環境
4. 驗收項目
5. 測試案例
6. 驗收結果
7. 問題追蹤
8. 附錄

### 章節說明

- **驗收目的**：驗收目標與驗收標準。
- **驗收範圍**：包含與不包含的驗收項目。
- **驗收環境**：測試環境規格、硬體、軟體、網路。
- **驗收項目**：以表格列出所有驗收項目與對應規格。
- **測試案例**：每個測試案例的步驟、預期結果、實際結果。
- **驗收結果**：驗收結論、通過率、未通過項目。
- **問題追蹤**：以表格列出問題、嚴重度、狀態、負責人。
- **附錄**：測試資料、測試腳本、簽名頁。
